1) Čo je tento projekt (1–3 vety)
Toto je single-page landing stránka v slovenčine pre službu „Čistá strecha z neba“ (čistenie striech dronom) s cieľom získať predobjednávky a vybrať zálohu 50 € cez Stripe. Frontend je hotový v jednom súbore `index.html` (inline CSS + JS), dokumentácia je v `PROJECT_HANDOVER.md` a `SITE_OVERVIEW.md`. Priečinok `_delivery/` je zrkadlový balík pre audit; kľúčové súbory sú identické s root verziami.

2) Ako funguje funnel krok za krokom (hero → CTA → form → backend → Stripe)
Krok 1: Používateľ príde na hero sekciu a klikne CTA „Začať objednávku“ (anchor `#objednavka`).
Krok 2: V objednávkovom formulári vyplní povinné polia `meno`, `adresa`, `email` (telefón a poznámka sú voliteľné).
Krok 3: JS validuje povinné polia a formát emailu; pri chybe zvýrazní polia červenou.
Krok 4: Pri validnom formulári JS odošle JSON `POST` na `ORDER_ENDPOINT = '/api/order'`.
Krok 5: Ak endpoint vráti 2xx, zobrazí sa success hláška, formulár sa resetne a otvorí sa Stripe Checkout (`STRIPE_CHECKOUT_URL`).
Krok 6: Ak endpoint zlyhá, Stripe sa neotvorí; používateľ vidí error hlášku a CTA sa znovu aktivuje.

3) Aktuálny stav: čo je hotové, čo je placeholder, čo je rizikové
Hotové:
- Kompletný one-pager obsah (hero, benefity, FAQ, formulár, recenzie, kroky, CTA, footer).
- Kontakty v UI sú konzistentné a sú klikateľné (`mailto:` + `tel:`).
- Validácia formulára + status box + disable/enable CTA počas requestu.

Placeholder / nedokončené:
- `/api/order` je placeholder endpoint bez backend implementácie v tomto workspaci.
- Stripe link je TEST (`https://buy.stripe.com/test_bIY14d0eW4TB3pu4gg`).
- Odkaz na obchodné podmienky smeruje na chýbajúci súbor `/obchodne-podmienky.pdf`.
- Nie je implementované analytics meranie (GA/Meta/pixel/event tracking).

Riziká:
- Bez backendu sa funnel zasekne pred Stripe (neprejde krok odoslania objednávky).
- Chýbajúce právne dokumenty znamenajú právne/compliance riziko pri ostrom spustení.
- Bez analytiky nebude možné riadiť optimalizáciu konverzií podľa dát.
- Dokument `_delivery/CONTACTS_SCAN.md` je neaktuálny (tvrdí, že tel odkazy neexistujú).

4) Závislosti potrebné na go-live (backend, Stripe prod link, legal docs)
- Funkčný backend endpoint `/api/order` (2xx pri úspechu, logovanie chýb, uloženie leadu, notifikácia e-mailom/CRM).
- Produkčný Stripe Checkout link + overenie platobného flowu (ideálne aj webhook pre potvrdenie platby).
- Reálny právny obsah: obchodné podmienky + GDPR/privacy dokumenty a korektné odkazy.
- Produkčný deployment a monitoring (chyby formulára/API, dostupnosť).
- Ujasnené procesy po platbe: potvrdenie, plánovanie termínu, fakturácia doplatku.

5) Kľúčové metriky na sledovanie (konverzné body)
- Návšteva stránky → klik hero CTA (`#objednavka`).
- Zobrazenie formulára (view) → začatie vyplňovania (input start).
- Odoslanie formulára (submit click) → validácia fail rate podľa poľa.
- `POST /api/order` success rate, error rate, latency.
- Otvorenie Stripe Checkout po úspešnom POST.
- Stripe payment success rate (záloha 50 €) a abandonment.
- Lead-to-booking rate (záloha → potvrdený termín).
- Čas od leadu po prvý kontakt (SLA) a po realizáciu.
